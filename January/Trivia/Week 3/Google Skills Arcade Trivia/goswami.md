# Google Cloud Trivia - Week 3

## Question 1: Connect Corporate Office to VPC
**You need to connect your corporate office to your Google Cloud VPC with a secure, encrypted connection. How will you establish this link?**

Select ONE answer that would be relevant:
- A) Use Cloud Functions
- B) Create a Cloud VPN Gateway and Tunnel
- C) Use a public HTTP website
- D) Use Google Drive

**Answer: B) Create a Cloud VPN Gateway and Tunnel**

## Question 2: Test VPN Bandwidth
**You have set up a VPN in Google Cloud and want to verify the speed using a benchmark tool. How will you generate traffic to test the bandwidth?**

Select ONE answer that would be relevant:
- A) Use Cloud Vision API
- B) Open a web browser and load a page
- C) Run the iperf tool between two VMs across the VPN
- D) Use the ping command only

**Answer: C) Run the iperf tool between two VMs across the VPN**

## Question 3: Storage Bucket Permissions
**You need to give a new team member permission to view files in a Google Cloud Storage bucket, but not delete them. How will you configure this?**

Select ONE answer that would be relevant:
- A) Create a new project for them
- B) Give them the Owner role
- C) Send them the files via email
- D) Add the user to the Storage Object Viewer role in IAM

**Answer: D) Add the user to the Storage Object Viewer role in IAM**

## Question 4: Authorize Python Script for BigQuery
**You have a Python script running on a Compute Engine VM in Google Cloud that needs to access BigQuery. How will you authorize this script securely?**

Select ONE answer that would be relevant:
- A) Assign a Service Account with BigQuery permissions to the VM
- B) Hardcode your username and password in the script
- C) Run the script on your local laptop instead
- D) Make the BigQuery dataset public to everyone

**Answer: A) Assign a Service Account with BigQuery permissions to the VM**

## Question 5: Managed Kubernetes
**You want to use Kubernetes on Google Cloud but want Google to handle all the node management and security patching. How will you do this?**

Select ONE answer that would be relevant:
- A) Install Kubernetes on a raw VM
- B) Use Cloud Run
- C) Select Standard mode and manage it yourself
- D) Select the Autopilot mode when creating a GKE cluster

**Answer: D) Select the Autopilot mode when creating a GKE cluster**

## Question 6: Check GKE Autopilot Nodes
**You have a GKE Autopilot cluster running. How will you check the status of the nodes that Google has automatically provisioned for you?**

Select ONE answer that would be relevant:
- A) You cannot see nodes in Autopilot
- B) Run kubectl get nodes
- C) Run docker ps
- D) Use Cloud SQL

**Answer: B) Run kubectl get nodes**
